MyWheels open mobility platform

Car Sharing , Bike Sharing, Boat sharing

We love to share

MyWheels is a not-for profit organization for shared mobility. Car Sharing and Bike Sharing are our favourites, but we also like boat sharing.

Beside sharing mobility, we love to share software. There are so many sharing platforms developing their own websites and App's. Together we can save a lot of effort and make a better and more sustainable platform. Feel free to contact us. We love to hear from you.

Ronald Haverman CTO MyWheels.nl

Vision

MyWheels provides an open standard and platform for car sharing and bike sharing. We started this initiative because sharing is getting popular and a lot of parties are busy developing software for sharing mobility. Software that has a lot of similarities with other sharing software. Apps with a lot off similarities.

We raised the question: How would the world be if we share this knowledge and software? How would it be if we work together to enable car sharing and bike sharing everywhere on this planet.

Central repository

We use Github as our central repository for our source code. Anyone can view our open source repositories, and with a free Github account, raise issues and contribute to our source code.

Help us with feature requests and bug fixes

We use Github Issues to track issues and features. Feel free to submit feature requests and raise bugs. We are happy with your help and feed-back. Please use our Github issue tracker for alle issues.

Contributions

We welcome contributions to all of our open source repositories. These contributions come in the form of a Pull Request, where you "fork" our repositories and then request that we "pull" your changes back into our repositories. You must have a Github account to make a contribution.

All contributors must first sign the MyWheels Individual Contributor License Agreement (CLA), which is based on the Google CLA, and provides the MyWheels team a license to re-distribute your contributions.

Whenever possible, please follow these guidelines for contributions:

  • Keep each pull request small and focused on a single feature or bug fix.
  • Familiarize yourself with the code base, and follow the formatting principles adhered to in the surrounding code.
  • Wherever possible, provide unit tests for your contributions.

OpenWheels platform & API access

Our shared mobility platform can be used as Software as a Service (SaaS) with the openwheels-frontend, or through the JSON-RPC API directly. Contact us to get access and receive the documentation.

Closed user group for platform source code

The platform source code (core and API) is not open source (yet) due to work to do, security and financial reasons. Because we love to share, the source code is available for a closed user group under (almost) the same conditions as open source. Please contact us for information and conditions how to apply. This might be an awesome way to develop your own version of the shared mobility platform.
